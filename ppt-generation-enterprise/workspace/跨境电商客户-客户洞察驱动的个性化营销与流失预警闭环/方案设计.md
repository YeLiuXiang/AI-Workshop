# 方案设计

本文档面向快速 vibe coding。重点是明确页面结构、展示模块、交互路径和前端实现建议，帮助团队直接开始搭建 营销增长与客户保留工作台 的原型壳子。

## 设计原则

- 优先交付可演示页面，不依赖真实后端
- 页面信息结构清晰，适合客户现场讲解
- 尽量使用假数据和静态状态完成业务路径演示
- 视觉风格建议：企业级 / 简洁 / 卡片式布局

## 页面结构

1. 客户洞察与客群分层面板
2. 营销文本、图像和页面生成区
3. 活动分析与流失预警行动区

## 模块布局

1. 客户洞察与客群分层面板 模块区
2. 营销文本、图像和页面生成区 模块区
3. 活动分析与流失预警行动区 模块区
4. 系统读取客户画像、交易行为、渠道反馈和历史活动表现
5. AI 完成客户分层，识别高价值客群、潜在流失客群和活动切入点
6. AI 自动生成营销文本、促销话术和多语言活动内容

## 交互路径

1. 选择目标国家站点、客群和营销目标
2. 查看 AI 输出的客群洞察、营销策略和风险提示
3. 一键生成文本、图片、动态页面和保留动作建议
4. 系统读取客户画像、交易行为、渠道反馈和历史活动表现
5. AI 完成客户分层，识别高价值客群、潜在流失客群和活动切入点

## 演示状态说明

- 首页展示关键概览卡片和主操作入口
- 详情页展示 AI 洞察、生成结果和建议动作的静态样例
- 用切换状态、Tab、Drawer 或 Mock 数据模拟关键动作反馈
- 所有指标、列表和详情内容都可以先用假数据驱动

## 推荐技术栈

- 前端建议：Next.js + Tailwind CSS
- 样式建议：组件化卡片布局 + 简洁图表占位 + 演示态弹层
- 数据建议：本地 JSON / mock 数据文件驱动页面展示

## 建议目录结构

- app/ 或 src/pages/：页面入口
- components/：卡片、表格、Tab、Drawer、图表占位组件
- data/：样例数据、页面假数据、切换状态配置
- docs/：保留本 PRD 和方案设计文档

## 快速开发建议

1. 确认一个国家站点和一个营销活动作为试点
2. 准备客户画像、活动结果和流失样本数据
3. 两周内完成工作台原型和验证口径
4. 基于方案设计文档快速搭建静态页面和样例数据

## 可直接复用的提示信息

- 目标用户：营销与 CRM 负责人
- 核心任务：围绕客户洞察、个性化营销、内容生成、动态页面搭建和流失预警，形成一个可复制的 AI 增长闭环。
- 原型形态：营销增长与客户保留工作台
- 原型价值：让营销团队从多系统切换和人工拼装，转向在一个界面里完成洞察、生成、上线和保留干预。
