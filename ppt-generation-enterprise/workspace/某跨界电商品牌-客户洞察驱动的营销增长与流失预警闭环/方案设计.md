# 方案设计

本文档面向快速 vibe coding。重点是明确页面结构、展示模块、交互路径和前端实现建议，帮助团队直接开始搭建 营销增长与客户保留工作台 的原型壳子。

## 设计原则

- 优先交付可演示页面，不依赖真实后端
- 页面信息结构清晰，适合客户现场讲解
- 尽量使用假数据和静态状态完成业务路径演示
- 视觉风格建议：企业级 / 增长工作台 / 营销运营看板

## 页面结构

1. 客户洞察与客群分层总览
2. 活动策略与内容生成页
3. 活动复盘与效果分析页
4. 流失预警与干预行动页

## 模块布局

1. 高价值客群与风险客群卡片
2. 营销文案与页面建议面板
3. 活动效果分析图区
4. 流失预警与跟进任务面板
5. 客户洞察与客群分层总览 模块区
6. 活动策略与内容生成页 模块区

## 交互路径

1. 切换国家站点和目标客群查看洞察
2. 生成个性化营销内容和页面建议
3. 查看活动复盘与下一轮优化建议
4. 触发高风险客户干预动作
5. 选择目标客群、活动目标和重点国家站点
6. 查看 AI 输出的客群洞察、营销策略和风险提示

## 交付件

- 方案设计：页面结构、模块布局、交互路径
- 架构设计：展示层、组件层、样例数据层
- 静态页面原型：可现场演示的前端壳子
- 样例数据包：支撑页面演示的 mock 数据

## 架构设计

- 展示层：页面壳子、看板卡片、列表表格、详情抽屉
- 交互层：筛选切换、建议查看、动作触发、结果回看
- 数据层：样例数据、页面状态配置、Mock 接口返回

## 演示状态说明

- 首页展示关键概览卡片和主操作入口
- 详情页展示 AI 洞察、生成结果和建议动作的静态样例
- 用切换状态、Tab、Drawer 或 Mock 数据模拟关键动作反馈
- 所有指标、列表和详情内容都可以先用假数据驱动

## 推荐技术栈

- 前端建议：Next.js + Tailwind CSS
- 样式建议：组件化卡片布局 + 简洁图表占位 + 演示态弹层
- 数据建议：本地 JSON / mock 数据文件驱动页面展示

## 建议目录结构

- app/ 或 src/pages/：页面入口
- components/：卡片、表格、Tab、Drawer、图表占位组件
- data/：样例数据、页面假数据、切换状态配置
- docs/：保留本 PRD 和方案设计文档

## 快速开发建议

1. 确认一个重点国家站点和一类目标客群作为试点
2. 准备客户画像、活动结果和流失样本数据
3. 两周内完成工作台原型与验证口径
4. 基于方案设计文档快速搭建静态页面和样例数据

## 可直接复用的提示信息

- 目标用户：营销负责人
- 核心任务：在一个工作台界面里完成客群洞察查看、活动内容生成、复盘分析和流失干预触发。
- 原型形态：营销增长与客户保留工作台
- 原型价值：让营销团队从多系统切换和人工拼装，转向在一个工作台里完成洞察、生成、复盘和干预。
