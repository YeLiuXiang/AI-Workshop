# 方案设计

本文档面向快速 vibe coding。重点是明确页面结构、展示模块、交互路径和前端实现建议，帮助团队直接开始搭建 客户运营与内容生成工作台 的原型壳子。

## 设计原则

- 优先交付可演示页面，不依赖真实后端
- 页面信息结构清晰，适合客户现场讲解
- 尽量使用假数据和静态状态完成业务路径演示
- 视觉风格建议：企业级 / 卡片式布局

## 页面结构

1. 客户洞察与客群分层总览
2. 营销文本与图片生成页
3. 动态页面编排页
4. 活动分析与流失预警页

## 模块布局

1. 客户分层与运营机会卡片
2. 营销文案生成面板
3. 营销配图生成面板
4. 动态页面预览与编辑区
5. 活动效果分析图区
6. 高风险客户预警与跟进行动面板

## 交互路径

1. 切换目标客群查看客户洞察
2. 生成营销文本和配图建议
3. 组合页面内容并预览活动页
4. 查看活动分析结果和下一轮优化建议
5. 触发高风险客户保留动作建议
6. 查看客户洞察并识别重点客群

## 交付件

- 方案设计：页面结构、模块布局、交互路径
- 架构设计：展示层、组件层、样例数据层
- 静态页面原型：可现场演示的前端壳子
- 样例数据包：支撑页面演示的 mock 数据

## 架构设计

- 展示层：页面壳子、看板卡片、列表表格、详情抽屉
- 交互层：筛选切换、建议查看、动作触发、结果回看
- 数据层：样例数据、页面状态配置、Mock 接口返回

## 演示状态说明

- 首页展示关键概览卡片和主操作入口
- 详情页展示 AI 洞察、生成结果和建议动作的静态样例
- 用切换状态、Tab、Drawer 或 Mock 数据模拟关键动作反馈
- 所有指标、列表和详情内容都可以先用假数据驱动

## 推荐技术栈

- 前端建议：Next.js + Tailwind CSS
- 样式建议：组件化卡片布局 + 简洁图表占位 + 演示态弹层
- 数据建议：本地 JSON / mock 数据文件驱动页面展示

## 建议目录结构

- app/ 或 src/pages/：页面入口
- components/：卡片、表格、Tab、Drawer、图表占位组件
- data/：样例数据、页面假数据、切换状态配置
- docs/：保留本 PRD 和方案设计文档

## 快速开发建议

1. 确认一个重点客群和一类典型活动作为试点
2. 准备客户画像、活动结果和流失样本数据
3. 两周内完成工作台原型与主流程演示
4. 基于方案设计文档快速搭建静态页面和样例数据

## 可直接复用的提示信息

- 目标用户：业务与数字化负责人
- 核心任务：查看客户洞察并联动营销与内容生成动作
- 原型形态：客户运营与内容生成工作台
- 原型价值：让业务与数字化负责人在一个工作台里看到从客户理解到内容生成再到客户保留的完整 AI 闭环。
