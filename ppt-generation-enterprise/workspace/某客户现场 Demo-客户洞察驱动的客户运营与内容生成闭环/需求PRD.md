# 需求PRD

本文档用于快速产出 某客户现场 Demo 在“客户洞察驱动的客户运营与内容生成闭环”场景下的轻量原型。目标是先完成可演示的前端壳子，而不是功能完整的系统。

## 项目概述

- 客户：某客户现场 Demo
- 场景：客户洞察驱动的客户运营与内容生成闭环
- 目标用户：业务与数字化负责人
- 原型名称：客户运营与内容生成工作台
- 原型模式：前端壳子优先

## 问题与目标

- 当前问题：客户洞察、营销动作和内容生成链路分散，缺少统一流程表达，导致客户运营判断、内容生产和后续保留动作难以形成闭环。
- 业务目标：快速形成一版可讲清业务闭环的 AI 场景方案，并先通过工作台原型验证客户洞察、内容生成和客户运营协同的直接价值。
- 原型目标：把客户洞察、营销内容生成、页面承载、活动复盘和流失预警集中到一个工作台中进行演示和协同
- 核心任务：查看客户洞察并联动营销与内容生成动作

## MVP 范围

1. 客户洞察与客群分层总览
2. 营销文本与图片生成页
3. 动态页面编排页
4. 活动分析与流失预警页

## 关键模块

1. 客户分层与运营机会卡片
2. 营销文案生成面板
3. 营销配图生成面板
4. 动态页面预览与编辑区
5. 活动效果分析图区
6. 高风险客户预警与跟进行动面板

## 关键动作

1. 切换目标客群查看客户洞察
2. 生成营销文本和配图建议
3. 组合页面内容并预览活动页
4. 查看活动分析结果和下一轮优化建议
5. 触发高风险客户保留动作建议
6. 查看客户洞察并识别重点客群

## 样例数据

1. 客户画像样例
2. 历史活动结果样例
3. 营销文案样例
4. 高风险客户清单
5. 场景步骤样例：基于客户数据了解客户特征和分层
6. 场景步骤样例：围绕目标客群策划个性化营销动作

## 交付清单

- 可演示的前端壳子页面
- 需求PRD.md
- 方案设计.md

## 非目标范围

- 不接真实营销自动化平台
- 不直接执行批量消息触达
- 不实现正式权限体系

## 验收标准

- 营销内容准备周期缩短 40%
- 内容与页面协同效率提升 50%
- 可以通过静态样例完整演示核心业务路径
