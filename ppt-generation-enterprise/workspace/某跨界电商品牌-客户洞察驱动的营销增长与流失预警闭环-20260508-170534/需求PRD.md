# 需求PRD

本文档用于快速产出 某跨界电商品牌 在“客户洞察驱动的营销增长与流失预警闭环”场景下的轻量原型。目标是先完成可演示的前端壳子，而不是功能完整的系统。

## 项目概述

- 客户：某跨界电商品牌
- 场景：客户洞察驱动的营销增长与流失预警闭环
- 目标用户：营销负责人
- 原型名称：营销增长与客户保留工作台

## 问题与目标

- 当前问题：客户画像、活动数据、内容生产和客户保留动作分散在不同环节，导致营销准备周期长、活动复盘慢、流失预警与干预滞后。
- 业务目标：把客户洞察、内容生成、活动执行、复盘分析和流失干预串成一个可复制的 AI 增长闭环，先验证对营销效率和客户保留的直接价值。
- 原型目标：把客户洞察、内容生产、活动分析和流失预警集中到一个可操作的工作台中
- 核心任务：在一个工作台界面里完成客群洞察查看、活动内容生成、复盘分析和流失干预触发。

## MVP 范围

1. 客户洞察与客群分层总览
2. 活动策略与内容生成页
3. 活动复盘与效果分析页
4. 流失预警与干预行动页

## 关键模块

1. 高价值客群与风险客群卡片
2. 营销文案与页面建议面板
3. 活动效果分析图区
4. 流失预警与跟进任务面板
5. 客户洞察与客群分层总览 模块区
6. 活动策略与内容生成页 模块区

## 关键动作

1. 切换国家站点和目标客群查看洞察
2. 生成个性化营销内容和页面建议
3. 查看活动复盘与下一轮优化建议
4. 触发高风险客户干预动作
5. 选择目标客群、活动目标和重点国家站点
6. 查看 AI 输出的客群洞察、营销策略和风险提示

## 样例数据

1. 客户画像样例
2. 活动结果样例
3. 高风险客户清单
4. 场景步骤样例：汇总客户画像、活动数据和渠道反馈，识别目标客群与重点活动机会
5. 场景步骤样例：营销团队基于经验制定活动策略与个性化内容方向

## 交付清单

- 需求PRD.md

## 非目标范围

- 不接真实营销自动化平台
- 不直接执行批量消息触达
- 不实现正式权限体系

## 验收标准

- 营销活动准备周期缩短 40%
- 内容与页面生产效率提升 50%
- 可以通过静态样例完整演示核心业务路径
