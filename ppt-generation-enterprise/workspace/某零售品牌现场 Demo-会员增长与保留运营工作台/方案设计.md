# 方案设计

本文档面向快速 vibe coding。重点是明确页面结构、展示模块、交互路径和前端实现建议，帮助团队直接开始搭建 会员增长与保留运营工作台 的原型壳子。

## 设计原则

- 优先交付可演示页面，不依赖真实后端
- 页面信息结构清晰，适合客户现场讲解
- 尽量使用假数据和静态状态完成业务路径演示
- 视觉风格建议：企业级 / 卡片式布局 / 增长运营

## 页面结构

1. 会员经营总览页
2. 客群洞察与分层页
3. 营销内容与活动页面生成页
4. 流失预警与保留动作页

## 模块布局

1. 会员分层看板
2. 重点客群机会卡片
3. 营销文案生成面板
4. 活动配图生成面板
5. 活动页面编排区
6. 高流失会员预警与跟进面板

## 交互路径

1. 切换目标会员客群
2. 生成营销文案
3. 生成配图与落地页草稿
4. 查看活动效果分析
5. 触发会员保留建议
6. 查看 AI 输出的客群洞察与风险提示

## 交付件

- 方案设计：页面结构、模块布局、交互路径
- 架构设计：展示层、组件层、样例数据层
- 静态页面原型：可现场演示的前端壳子
- 样例数据包：支撑页面演示的 mock 数据

## 架构设计

- 展示层：页面壳子、看板卡片、列表表格、详情抽屉
- 交互层：筛选切换、建议查看、动作触发、结果回看
- 数据层：样例数据、页面状态配置、Mock 接口返回

## 演示状态说明

- 首页展示关键概览卡片和主操作入口
- 详情页展示 AI 洞察、生成结果和建议动作的静态样例
- 用切换状态、Tab、Drawer 或 Mock 数据模拟关键动作反馈
- 所有指标、列表和详情内容都可以先用假数据驱动

## 推荐技术栈

- 前端建议：Next.js + Tailwind CSS
- 样式建议：组件化卡片布局 + 简洁图表占位 + 演示态弹层
- 数据建议：本地 JSON / mock 数据文件驱动页面展示

## 建议目录结构

- app/ 或 src/pages/：页面入口
- components/：卡片、表格、Tab、Drawer、图表占位组件
- data/：样例数据、页面假数据、切换状态配置
- docs/：保留本 PRD 和方案设计文档

## 快速开发建议

1. 确认一个重点会员客群和一类活动场景作为试点
2. 准备会员画像、活动结果和高流失会员样本数据
3. 两周内完成工作台原型与验证口径
4. 基于方案设计文档快速搭建静态页面和样例数据

## 可直接复用的提示信息

- 目标用户：会员运营负责人
- 核心任务：查看重点会员客群并联动营销内容与保留动作
- 原型形态：会员增长与保留运营工作台
- 原型价值：让会员运营团队从多系统切换和人工拼装，转向在一个工作台里完成洞察、生成、复盘和干预。
