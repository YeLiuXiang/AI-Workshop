# 方案设计

本文档面向快速 vibe coding。重点是明确页面结构、展示模块、交互路径和前端实现建议，帮助团队直接开始搭建 电子产品售后维修问答工作台 的原型壳子。

## 设计原则

- 优先交付可演示页面，不依赖真实后端
- 页面信息结构清晰，适合客户现场讲解
- 尽量使用假数据和静态状态完成业务路径演示
- 视觉风格建议：企业级 / 卡片式布局

## 页面结构

1. 维修问答首页
2. 问题分析详情页
3. 处理建议与升级动作页

## 模块布局

1. 问题输入模块
2. 知识检索模块
3. AI 问答模块
4. 处理建议模块
5. 工单协同模块
6. 维修问答首页 模块区

## 交互路径

1. 输入故障问题并发起检索
2. 查看 AI 推荐答案和历史案例
3. 生成处理建议或触发升级协同
4. 输入产品型号、故障现象或工单编号
5. 查看 AI 检索出的相关知识、历史案例和推荐判断
6. 一键生成回复话术、排障建议和升级动作

## 交付件

- 方案设计：页面结构、模块布局、交互路径
- 架构设计：展示层、组件层、样例数据层
- 静态页面原型：可现场演示的前端壳子
- 样例数据包：支撑页面演示的 mock 数据

## 架构设计

- 展示层：页面壳子、看板卡片、列表表格、详情抽屉
- 交互层：筛选切换、建议查看、动作触发、结果回看
- 数据层：样例数据、页面状态配置、Mock 接口返回

## 演示状态说明

- 首页展示关键概览卡片和主操作入口
- 详情页展示 AI 洞察、生成结果和建议动作的静态样例
- 用切换状态、Tab、Drawer 或 Mock 数据模拟关键动作反馈
- 所有指标、列表和详情内容都可以先用假数据驱动

## 推荐技术栈

- 前端建议：Next.js + Tailwind CSS
- 样式建议：组件化卡片布局 + 简洁图表占位 + 演示态弹层
- 数据建议：本地 JSON / mock 数据文件驱动页面展示

## 建议目录结构

- app/ 或 src/pages/：页面入口
- components/：卡片、表格、Tab、Drawer、图表占位组件
- data/：样例数据、页面假数据、切换状态配置
- docs/：保留本 PRD 和方案设计文档

## 快速开发建议

1. 确认试点产品范围和高频故障问题清单
2. 整理维修手册、FAQ 和历史工单样本
3. 两周内完成问答工作台原型和首轮效果验证
4. 基于方案设计文档快速搭建静态页面和样例数据

## 可直接复用的提示信息

- 目标用户：售后人员
- 核心任务：快速检索维修知识并生成可执行的处理建议
- 原型形态：售后维修问答工作台
- 原型价值：让售后人员从人工翻资料和反复请教专家，转向在一个工作台中完成检索、判断和处理动作。
