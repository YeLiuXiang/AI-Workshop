# 需求PRD

本文档用于快速产出 某客户现场 Demo 在“客户运营与内容生成工作台”场景下的轻量原型。目标是先完成可演示的前端壳子，而不是功能完整的系统。

## 项目概述

- 客户：某客户现场 Demo
- 场景：客户运营与内容生成工作台
- 目标用户：业务与数字化负责人
- 原型名称：客户运营与内容生成工作台
- 原型模式：前端壳子优先

## 问题与目标

- 当前问题：客户洞察、营销动作和内容生成链路分散，缺少统一流程表达。
- 业务目标：希望快速形成一版可讲清业务闭环的 AI 场景方案。
- 原型目标：希望快速形成一版可讲清业务闭环的 AI 场景方案。
- 核心任务：查看客户洞察并联动营销与内容生成动作

## MVP 范围

1. 客户运营与内容生成工作台总览
2. 分析详情页
3. 建议与动作页
4. 协同与复盘页

## 关键模块

1. 客户运营与内容生成工作台总览 模块区
2. 分析详情页 模块区
3. 建议与动作页 模块区
4. 协同与复盘页 模块区
5. 个性化营销
6. 生成或增强文本

## 关键动作

1. 查看核心业务风险或机会分布
2. 读取 AI 给出的判断与建议
3. 触发协同动作并回看处理结果
4. 个性化营销
5. 生成或增强文本

2. 场景步骤样例：个性化营销

## 交付清单

## 非目标范围

- 不接真实后端接口
- 不实现真实业务流程提交
- 不做权限体系和数据持久化
- 不做生产级异常处理

## 验收标准

- 核心判断时间缩短 30%
- 协同响应时间缩短 25%
- 可以通过静态样例完整演示核心业务路径
