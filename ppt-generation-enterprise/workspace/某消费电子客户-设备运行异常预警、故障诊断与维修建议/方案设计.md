# 方案设计

本文档面向快速 vibe coding。重点是明确页面结构、展示模块、交互路径和前端实现建议，帮助团队直接开始搭建 设备健康与维修决策工作台 的原型壳子。

## 设计原则

- 优先交付可演示页面，不依赖真实后端
- 页面信息结构清晰，适合客户现场讲解
- 尽量使用假数据和静态状态完成业务路径演示
- 视觉风格建议：企业级 / 运营看板 / 卡片式布局 / 数据监控感

## 页面结构

1. 设备健康总览页
2. 异常诊断详情页
3. 维修建议与工单动作页
4. 知识库与历史案例页

## 模块布局

1. 设备健康总览页 模块区
2. 异常诊断详情页 模块区
3. 维修建议与工单动作页 模块区
4. 知识库与历史案例页 模块区
5. 接入并汇总设备运行日志、功耗、温感、告警记录和历史维修工单
6. AI 对多源数据进行异常检测、趋势分析和风险评分，判断死机、老化或故障概率

## 交互路径

1. 查看高风险设备列表、健康评分和异常原因摘要
2. 点击设备详情，查看日志、功耗和温感趋势及 AI 诊断结论
3. 一键生成维修建议、备件建议、客服告知话术和工单动作
4. 对高风险设备直接触发派修、回访或人工复核
5. 接入并汇总设备运行日志、功耗、温感、告警记录和历史维修工单
6. AI 对多源数据进行异常检测、趋势分析和风险评分，判断死机、老化或故障概率

## 交付件

- 方案设计：页面结构、模块布局、交互路径
- 架构设计：展示层、组件层、样例数据层
- 静态页面原型：可现场演示的前端壳子
- 样例数据包：支撑页面演示的 mock 数据

## 架构设计

- 展示层：页面壳子、看板卡片、列表表格、详情抽屉
- 交互层：筛选切换、建议查看、动作触发、结果回看
- 数据层：样例数据、页面状态配置、Mock 接口返回

## 演示状态说明

- 首页展示关键概览卡片和主操作入口
- 详情页展示 AI 洞察、生成结果和建议动作的静态样例
- 用切换状态、Tab、Drawer 或 Mock 数据模拟关键动作反馈
- 所有指标、列表和详情内容都可以先用假数据驱动

## 推荐技术栈

- 前端建议：Next.js + Tailwind CSS
- 样式建议：组件化卡片布局 + 简洁图表占位 + 演示态弹层
- 数据建议：本地 JSON / mock 数据文件驱动页面展示

## 建议目录结构

- app/ 或 src/pages/：页面入口
- components/：卡片、表格、Tab、Drawer、图表占位组件
- data/：样例数据、页面假数据、切换状态配置
- docs/：保留本 PRD 和方案设计文档

## 快速开发建议

1. 确认试点设备型号、故障类型和可用历史数据样本
2. 整理日志、功耗、温感和维修工单的字段口径
3. 两到三周内完成工作台原型和首轮效果验证
4. 基于方案设计文档快速搭建静态页面和样例数据

## 可直接复用的提示信息

- 目标用户：售后服务负责人、维修工程师
- 核心任务：快速识别高风险设备并获得可执行的诊断与维修建议
- 原型形态：设备健康运营与维修决策工作台
- 原型价值：让售后团队从事后排障转向事前预警与标准化处置，缩短故障定位时间并提升维修决策质量。
