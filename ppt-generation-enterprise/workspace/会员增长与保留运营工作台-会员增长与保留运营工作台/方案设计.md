# 方案设计

本文档面向快速 vibe coding。重点是明确页面结构、展示模块、交互路径和前端实现建议，帮助团队直接开始搭建 会员增长与保留运营工作台 的原型壳子。

## 设计原则

- 优先交付可演示页面，不依赖真实后端
- 页面信息结构清晰，适合客户现场讲解
- 尽量使用假数据和静态状态完成业务路径演示
- 视觉风格建议：企业级 / 卡片式布局 / 增长运营 / 数据洞察

## 页面结构

1. 运营首页
2. 会员洞察页
3. 内容生成页
4. 保留动作与复盘页

## 模块布局

1. 会员分层总览卡片
2. 高价值会员与流失风险榜单
3. 活动策略与文案生成区
4. 页面内容预览区
5. 保留动作推荐区
6. 活动效果复盘区

## 交互路径

1. 筛选目标会员客群
2. 查看 AI 洞察与风险标签
3. 生成营销文案和页面内容
4. 确认并下发保留动作建议
5. 查看活动反馈与复盘结果
6. 查看会员分层与风险洞察

## 交付件

- 方案设计：页面结构、模块布局、交互路径
- 架构设计：展示层、组件层、样例数据层
- 静态页面原型：可现场演示的前端壳子
- 样例数据包：支撑页面演示的 mock 数据

## 架构设计

- 展示层：页面壳子、看板卡片、列表表格、详情抽屉
- 交互层：筛选切换、建议查看、动作触发、结果回看
- 数据层：样例数据、页面状态配置、Mock 接口返回

## 演示状态说明

- 首页展示关键概览卡片和主操作入口
- 详情页展示 AI 洞察、生成结果和建议动作的静态样例
- 用切换状态、Tab、Drawer 或 Mock 数据模拟关键动作反馈
- 所有指标、列表和详情内容都可以先用假数据驱动

## 推荐技术栈

- 前端建议：Next.js + Tailwind CSS
- 样式建议：组件化卡片布局 + 简洁图表占位 + 演示态弹层
- 数据建议：本地 JSON / mock 数据文件驱动页面展示

## 建议目录结构

- app/ 或 src/pages/：页面入口
- components/：卡片、表格、Tab、Drawer、图表占位组件
- data/：样例数据、页面假数据、切换状态配置
- docs/：保留本 PRD 和方案设计文档

## 快速开发建议

1. 明确首批会员数据与活动样例范围
2. 确定 MVP 页面结构与演示样例数据
3. 启动 2 到 4 周 POC，验证洞察、生成与保留联动效果
4. 基于方案设计文档快速搭建静态页面和样例数据

## 可直接复用的提示信息

- 目标用户：会员运营负责人
- 核心任务：围绕会员增长与保留活动，快速完成客群识别、内容生成和保留动作联动演示。
- 原型形态：工作台
- 原型价值：先用前端壳子把会员洞察、内容生产和保留动作联动价值讲清楚，再决定是否扩展到真实系统对接。
