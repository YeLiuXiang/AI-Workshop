# 需求PRD

本文档用于快速产出 会员增长与保留运营工作台 在“会员增长与保留运营工作台”场景下的轻量原型。目标是先完成可演示的前端壳子，而不是功能完整的系统。

## 项目概述

- 客户：会员增长与保留运营工作台
- 场景：会员增长与保留运营工作台
- 目标用户：会员运营负责人
- 原型名称：会员增长与保留运营工作台
- 原型模式：前端壳子优先

## 问题与目标

- 当前问题：会员分层、营销内容生产、活动落地和流失预警分散在多个系统与团队中，响应慢且协同成本高。
- 业务目标：先以可演示工作台验证客户洞察、内容生成和保留动作联动价值，缩短运营响应周期并提升会员增长与留存效果。
- 原型目标：让会员运营负责人在一个可视化工作台内完成客群识别、内容生成、活动联动和流失干预演示。
- 核心任务：围绕会员增长与保留活动，快速完成客群识别、内容生成和保留动作联动演示。

## MVP 范围

1. 运营首页
2. 会员洞察页
3. 内容生成页
4. 保留动作与复盘页

## 关键模块

1. 会员分层总览卡片
2. 高价值会员与流失风险榜单
3. 活动策略与文案生成区
4. 页面内容预览区
5. 保留动作推荐区
6. 活动效果复盘区

## 关键动作

1. 筛选目标会员客群
2. 查看 AI 洞察与风险标签
3. 生成营销文案和页面内容
4. 确认并下发保留动作建议
5. 查看活动反馈与复盘结果
6. 查看会员分层与风险洞察

## 样例数据

1. 会员分层标签与价值评分样例
2. 历史活动效果与渠道反馈样例
3. 流失风险会员名单样例
4. AI 生成营销文案与页面草案样例
5. 场景步骤样例：制定会员增长或保留活动目标
6. 场景步骤样例：人工汇总客户画像、活动数据和渠道反馈

## 交付清单

- 可演示的前端壳子页面
- 需求PRD.md
- 方案设计.md

## 非目标范围

- 不接入真实营销渠道发送能力
- 不实现真实会员数据写回
- 不建设完整权限体系和生产级流程编排

## 验收标准

- 活动准备周期缩短 30%
- 高风险会员识别时效提升 50%
- 可以通过静态样例完整演示核心业务路径
