# 方案设计

本文档面向快速 vibe coding。重点是明确页面结构、展示模块、交互路径和前端实现建议，帮助团队直接开始搭建 企业知识问答与培训助手工作台 的原型壳子。

## 设计原则

- 优先交付可演示页面，不依赖真实后端
- 页面信息结构清晰，适合客户现场讲解
- 尽量使用假数据和静态状态完成业务路径演示
- 视觉风格建议：企业级 / 卡片式布局

## 页面结构

1. 知识总览页
2. 智能问答页
3. 培训内容整理页
4. 知识回写页

## 模块布局

1. 知识来源接入概览
2. 问题输入与问答结果区
3. 培训摘要与学习路径推荐
4. 高频问题与知识热度看板
5. 专家补充与知识回写面板
6. 知识总览页 模块区

## 交互路径

1. 搜索或提问获取知识答案
2. 查看相关制度文档和项目案例片段
3. 生成培训摘要和学习建议
4. 标记高价值问答并回写知识库
5. 进入工作台查看重点知识主题和推荐培训内容
6. 输入问题或选择场景，获取 AI 问答结果与参考资料

## 交付件

- 方案设计：页面结构、模块布局、交互路径
- 架构设计：展示层、组件层、样例数据层
- 静态页面原型：可现场演示的前端壳子
- 样例数据包：支撑页面演示的 mock 数据

## 架构设计

- 展示层：页面壳子、看板卡片、列表表格、详情抽屉
- 交互层：筛选切换、建议查看、动作触发、结果回看
- 数据层：样例数据、页面状态配置、Mock 接口返回

## 演示状态说明

- 首页展示关键概览卡片和主操作入口
- 详情页展示 AI 洞察、生成结果和建议动作的静态样例
- 用切换状态、Tab、Drawer 或 Mock 数据模拟关键动作反馈
- 所有指标、列表和详情内容都可以先用假数据驱动

## 推荐技术栈

- 前端建议：Next.js + Tailwind CSS
- 样式建议：组件化卡片布局 + 简洁图表占位 + 演示态弹层
- 数据建议：本地 JSON / mock 数据文件驱动页面展示

## 建议目录结构

- app/ 或 src/pages/：页面入口
- components/：卡片、表格、Tab、Drawer、图表占位组件
- data/：样例数据、页面假数据、切换状态配置
- docs/：保留本 PRD 和方案设计文档

## 快速开发建议

1. 确认首批接入的制度文档、项目案例和 FAQ 范围
2. 选择一个高频培训主题和一类典型答疑问题作为试点
3. 输出工作台原型范围、样例数据和 POC 验证口径
4. 基于方案设计文档快速搭建静态页面和样例数据

## 可直接复用的提示信息

- 目标用户：培训与知识管理负责人
- 核心任务：快速生成一版知识问答与培训方案，验证统一入口、问答推荐和培训内容整理的价值。
- 原型形态：企业知识问答与培训助手工作台
- 原型价值：让培训与知识管理负责人先在一个统一工作台里看到 AI 对知识问答、培训整理和组织沉淀的直接价值。
