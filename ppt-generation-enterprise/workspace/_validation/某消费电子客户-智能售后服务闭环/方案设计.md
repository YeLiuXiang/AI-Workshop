# 方案设计

本文档面向快速 vibe coding。重点是明确页面结构、展示模块、交互路径和前端实现建议，帮助团队直接开始搭建 智能售后服务工作台 的原型壳子。

## 设计原则

- 优先交付可演示页面，不依赖真实后端
- 页面信息结构清晰，适合客户现场讲解
- 尽量使用假数据和静态状态完成业务路径演示
- 视觉风格建议：企业级 / 简洁 / 卡片式布局

## 页面结构

1. 设备状态与客户上下文面板
2. AI 诊断与知识召回面板
3. 回复建议与工单动作面板

## 模块布局

1. 设备状态与客户上下文面板 模块区
2. AI 诊断与知识召回面板 模块区
3. 回复建议与工单动作面板 模块区
4. 系统读取设备状态、历史工单与 FAQ 等知识资料
5. AI 判断问题类型、风险等级和处理优先级
6. AI 生成客户回复与排障建议，支持一线客服快速响应

## 交互路径

1. 接收报修请求后自动汇总设备上下文与历史服务记录
2. 展示 AI 判断的问题类型、优先级与推荐处置路径
3. 一键生成回复建议、排障动作和升级工单
4. 系统读取设备状态、历史工单与 FAQ 等知识资料
5. AI 判断问题类型、风险等级和处理优先级

## 交付件

- 方案设计：页面结构、模块布局、交互路径
- 架构设计：展示层、组件层、样例数据层
- 静态页面原型：可现场演示的前端壳子
- 样例数据包：支撑页面演示的 mock 数据

## 架构设计

- 展示层：页面壳子、看板卡片、列表表格、详情抽屉
- 交互层：筛选切换、建议查看、动作触发、结果回看
- 数据层：样例数据、页面状态配置、Mock 接口返回

## 演示状态说明

- 首页展示关键概览卡片和主操作入口
- 详情页展示 AI 洞察、生成结果和建议动作的静态样例
- 用切换状态、Tab、Drawer 或 Mock 数据模拟关键动作反馈
- 所有指标、列表和详情内容都可以先用假数据驱动

## 推荐技术栈

- 前端建议：Next.js + Tailwind CSS
- 样式建议：组件化卡片布局 + 简洁图表占位 + 演示态弹层
- 数据建议：本地 JSON / mock 数据文件驱动页面展示

## 建议目录结构

- app/ 或 src/pages/：页面入口
- components/：卡片、表格、Tab、Drawer、图表占位组件
- data/：样例数据、页面假数据、切换状态配置
- docs/：保留本 PRD 和方案设计文档

## 快速开发建议

1. 确认试点设备范围与高频报修问题样本
2. 梳理 FAQ、历史工单与设备状态字段映射
3. 两周内输出工作台原型与首轮 POC 指标
4. 基于方案设计文档快速搭建静态页面和样例数据

## 可直接复用的提示信息

- 目标用户：售后负责人
- 核心任务：以设备状态驱动的方式提升问题识别效率、回复质量和服务闭环能力。
- 原型形态：智能售后服务工作台
- 原型价值：让客服从人工翻资料、人工找专家，转向在一个工作台里完成识别、回复、排障和动作触发。
