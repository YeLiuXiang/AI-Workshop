# 方案设计

本文档面向快速 vibe coding。重点是明确页面结构、展示模块、交互路径和前端实现建议，帮助团队直接开始搭建 库存与履约控制塔 的原型壳子。

## 设计原则

- 优先交付可演示页面，不依赖真实后端
- 页面信息结构清晰，适合客户现场讲解
- 尽量使用假数据和静态状态完成业务路径演示
- 视觉风格建议：企业级 / 控制塔 / 运营看板

## 页面结构

1. 全局风险总览
2. 区域库存与订单分析页
3. 补货调拨建议页
4. 协同任务与复盘页

## 模块布局

1. 区域风险地图
2. 缺货与积压预警卡片
3. 履约异常解释区
4. 补货调拨建议列表
5. 协同任务面板
6. 全局风险总览 模块区

## 交互路径

1. 切换区域查看库存与订单风险
2. 查看 AI 给出的补货调拨建议
3. 触发区域协同任务
4. 回看执行结果与复盘口径
5. 查看区域订单、库存和履约风险分布
6. 读取 AI 给出的补货、调拨和仓配建议

## 交付件

- 方案设计：页面结构、模块布局、交互路径
- 架构设计：展示层、组件层、样例数据层
- 静态页面原型：可现场演示的前端壳子
- 样例数据包：支撑页面演示的 mock 数据

## 架构设计

- 展示层：页面壳子、看板卡片、列表表格、详情抽屉
- 交互层：筛选切换、建议查看、动作触发、结果回看
- 数据层：样例数据、页面状态配置、Mock 接口返回

## 演示状态说明

- 首页展示关键概览卡片和主操作入口
- 详情页展示 AI 洞察、生成结果和建议动作的静态样例
- 用切换状态、Tab、Drawer 或 Mock 数据模拟关键动作反馈
- 所有指标、列表和详情内容都可以先用假数据驱动

## 推荐技术栈

- 前端建议：Next.js + Tailwind CSS
- 样式建议：组件化卡片布局 + 简洁图表占位 + 演示态弹层
- 数据建议：本地 JSON / mock 数据文件驱动页面展示

## 建议目录结构

- app/ 或 src/pages/：页面入口
- components/：卡片、表格、Tab、Drawer、图表占位组件
- data/：样例数据、页面假数据、切换状态配置
- docs/：保留本 PRD 和方案设计文档

## 快速开发建议

1. 确认试点区域和样本订单库存数据
2. 选择一个高波动品类作为试点
3. 输出控制塔原型范围和验证口径
4. 基于方案设计文档快速搭建静态页面和样例数据

## 可直接复用的提示信息

- 目标用户：供应链负责人
- 核心任务：在一个控制塔界面里完成库存履约风险判断、建议查看和协同任务触发。
- 原型形态：库存与履约控制塔
- 原型价值：让运营团队从人工拼表和分散协同，转向在控制塔里统一判断风险、建议和动作优先级。
